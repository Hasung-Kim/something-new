# Multi-Keyword YouTube Feed — Spec

## 개요

사용자가 관심 키워드를 여러 개 등록하면, YouTube 영상을 키워드별 컬럼으로
병렬 표시한다. YouTube를 한 키워드씩 반복 검색하는 불편 없이 여러 관심사를
한 화면에서 동시에 탐색할 수 있다.

## 범위

### 포함
- 키워드 등록 / 삭제 / 순서 변경 (상단 헤더)
- 키워드별 최신(관련도순) YouTube 영상 최대 8개
- 데스크탑: 키워드별 컬럼 레이아웃
- 모바일: 키워드별 탭 전환 레이아웃
- 영상 카드: 썸네일 / 제목 / 채널명 / 업로드 날짜
- AI 한 줄 요약 (카드별 버튼 클릭 시 생성)
- 페이지 새로고침 후 키워드 유지

### 제외
- 로그인 / 회원가입 — 핵심 가치 검증 전 불필요
- 영상 필터 (날짜 범위, 길이, 조회수) — MVP 이후 사용 패턴 확인 후 결정
- 시청 기록 / 본 영상 제외 기능 — 저장소 설계 복잡도
- 자동 새로고침 / 알림 — 수동 새로고침으로 충분
- 공유 / 협업 기능

## 시나리오

### 1. 첫 방문 — 키워드 없음

- **Given** 브라우저에 저장된 키워드가 없다
- **When** 사용자가 사이트를 방문한다
- **Then** 헤더 입력란이 포커스되고, 콘텐츠 영역에 예시 키워드 칩 목록이 표시된다

성공 기준:
- [ ] 예시 키워드 칩이 3개 이상 표시된다
- [ ] 예시 키워드 칩 클릭 시 해당 텍스트가 헤더 입력란에 채워진다
- [ ] 영상 컬럼은 하나도 표시되지 않는다

---

### 2. 키워드 추가

- **Given** 헤더 입력란이 비어 있다
- **When** "React"를 입력하고 Enter 키를 누르거나 "+추가" 버튼을 클릭한다
- **Then** "React ×" 칩이 헤더에 나타나고, 콘텐츠 영역에 "React" 컬럼이 추가되며 입력란은 비워진다

성공 기준:
- [ ] "React" 칩이 헤더에 표시된다
- [ ] "React" 컬럼이 콘텐츠 영역에 추가된다
- [ ] 입력란이 빈 상태로 초기화된다
- [ ] 빈 문자열 입력 시 칩이 추가되지 않는다
- [ ] 공백만 있는 입력 시 칩이 추가되지 않는다
- [ ] 이미 존재하는 키워드 재입력 시 중복 칩이 생기지 않는다

---

### 3. 키워드 최대 개수 초과

- **Given** 키워드 6개가 등록되어 있다
- **When** 사용자가 7번째 키워드를 입력하고 추가를 시도한다
- **Then** "최대 6개까지 추가할 수 있습니다" 메시지가 표시되고 키워드는 추가되지 않는다

성공 기준:
- [ ] 키워드가 6개일 때 "+추가" 버튼이 비활성화되거나 에러 메시지가 표시된다
- [ ] 7번째 키워드 칩이 추가되지 않는다
- [ ] 기존 6개 컬럼은 그대로 유지된다

---

### 4. 키워드 삭제

- **Given** "React", "주식" 키워드가 등록되어 있다
- **When** "React" 칩의 × 버튼을 클릭한다
- **Then** "React" 칩이 헤더에서 사라지고 "React" 컬럼이 콘텐츠 영역에서 제거된다

성공 기준:
- [ ] "React" 칩이 헤더에서 사라진다
- [ ] "React" 컬럼이 콘텐츠 영역에서 제거된다
- [ ] "주식" 컬럼은 그대로 유지된다

---

### 5. 키워드 순서 변경

- **Given** "React", "주식", "운동" 순서로 키워드가 등록되어 있다
- **When** "주식" 칩의 왼쪽(←) 화살표 버튼을 클릭한다
- **Then** 헤더와 콘텐츠 영역 모두 "주식", "React", "운동" 순서로 변경된다

성공 기준:
- [ ] 헤더 칩 순서가 "주식", "React", "운동"으로 바뀐다
- [ ] 콘텐츠 영역 컬럼 순서가 동일하게 바뀐다
- [ ] 맨 왼쪽 키워드의 ← 버튼은 비활성화된다
- [ ] 맨 오른쪽 키워드의 → 버튼은 비활성화된다

---

### 6. 영상 목록 로딩

- **Given** 사용자가 "React" 키워드를 방금 추가했다
- **When** "React" 컬럼이 화면에 나타난다
- **Then** 로딩 스켈레톤이 표시되다가, 관련도순 YouTube 영상 최대 8개가 썸네일·제목·채널명·업로드 날짜와 함께 표시된다

성공 기준:
- [ ] 로딩 중 스켈레톤 UI가 표시된다
- [ ] 영상 카드가 1개 이상 최대 8개까지 표시된다
- [ ] 각 카드에 썸네일 이미지가 표시된다
- [ ] 각 카드에 영상 제목이 표시된다
- [ ] 각 카드에 채널명이 표시된다
- [ ] 각 카드에 업로드 날짜가 표시된다
- [ ] 각 카드에 "요약 보기" 버튼이 표시된다

---

### 7. 영상 클릭

- **Given** 영상 카드들이 표시되어 있다
- **When** 특정 영상 카드를 클릭한다
- **Then** 새 탭에서 해당 YouTube 영상 페이지가 열린다

성공 기준:
- [ ] 현재 탭은 그대로 유지된다
- [ ] 새 탭이 열리며 해당 YouTube 동영상 URL로 이동한다

---

### 8. AI 한 줄 요약 생성

- **Given** 영상 카드가 표시되어 있고 "요약 보기" 버튼이 있다
- **When** "요약 보기" 버튼을 클릭한다
- **Then** 버튼이 로딩 상태로 바뀌다가, 한 줄 요약 텍스트가 카드에 표시된다. 이후 버튼은 사라진다.

성공 기준:
- [ ] 클릭 직후 버튼이 로딩 상태(스피너 또는 텍스트 변경)로 바뀐다
- [ ] 요약 생성 후 한 줄 텍스트가 카드에 나타난다
- [ ] 요약 표시 후 "요약 보기" 버튼은 더 이상 보이지 않는다
- [ ] 요약 생성 실패 시 버튼이 원래 상태로 돌아온다

---

### 9. YouTube API 에러

- **Given** "React", "주식" 컬럼이 표시되어 있다
- **When** "React" 키워드의 YouTube API 호출이 실패한다
- **Then** "React" 컬럼 내에 에러 메시지와 "다시 시도" 버튼이 표시된다. "주식" 컬럼은 정상 표시된다.

성공 기준:
- [ ] "React" 컬럼에 에러 메시지가 표시된다
- [ ] "React" 컬럼에 "다시 시도" 버튼이 표시된다
- [ ] "다시 시도" 버튼 클릭 시 해당 키워드 영상을 재요청한다
- [ ] "주식" 컬럼은 영향받지 않는다

---

### 10. 페이지 새로고침 후 키워드 유지

- **Given** "React", "주식", "운동" 키워드가 이 순서로 등록되어 있다
- **When** 페이지를 새로고침한다
- **Then** "React", "주식", "운동" 컬럼이 동일한 순서로 표시된다

성공 기준:
- [ ] 새로고침 후 동일한 키워드가 동일한 순서로 유지된다
- [ ] 키워드별 영상 로딩이 다시 시작된다

---

### 11. 모바일 뷰 (탭 전환)

- **Given** 모바일 화면(너비 768px 미만)에서 "React", "주식", "운동" 키워드가 등록되어 있다
- **When** 사용자가 페이지를 본다
- **Then** 컬럼 레이아웃 대신 키워드별 탭 UI가 표시되고, 탭 클릭으로 해당 키워드 영상을 볼 수 있다

성공 기준:
- [ ] 768px 미만에서 탭 UI가 표시된다
- [ ] 탭 클릭 시 해당 키워드의 영상 카드 목록이 표시된다
- [ ] 현재 선택된 탭이 시각적으로 구분된다

## 불변 규칙

### 보안
- YouTube API 키와 Anthropic API 키는 어떤 클라이언트 응답에도 포함되지 않는다

### 캐싱
- 동일 키워드는 마지막 API 호출로부터 1시간 이내 재요청 시 캐시된 영상 목록을 반환한다
- 캐시된 결과는 새 영상 목록과 동일한 UI로 표시된다 (캐시 여부 별도 표시 없음)
